## 결과 분석

**스캔 행수 99.96% 감소, p95 latency 23% 개선**

인덱스 없는 버전의 실제 동작:
- `movie_person` 전체 50,020행을 순차 스캔
- 그 중 `person_id`가 일치하는 20행만 반환
- 반환 데이터는 동일하지만 불필요한 50,000행 스캔 비용 발생

**TPS 차이가 미미한 이유**
sleep(1)로 VU당 1초에 1요청으로 제한되어 TPS 상한이 50으로 고정됨.
RDS 버퍼 풀에 데이터가 캐싱되어 풀스캔도 디스크 I/O 없이 처리됨.

**max 응답시간이 크게 개선된 이유**
인덱스 없는 경우 경합이 몰리는 순간 풀스캔 비용이 누적되어 최악 케이스(max)가 크게 튀지만,
인덱스가 있으면 접근 범위가 20행으로 고정되어 최악 케이스도 안정적으로 유지됨.
